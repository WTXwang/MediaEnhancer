# 影音智增强系统  
## 项目说明书（最终版）

---

### 一、项目概述

**影音智增强系统** 是一款集**本地影音资产管理**与**AI实时视觉增强**于一体的 Windows 桌面应用。它旨在解决普通用户在观看或整理低光照、有雾、有雨等低质量影像时“看不清、理还乱”的痛点。系统设计支持多种图像增强方法，初期以便于开发的**线性拉伸**作为基础实现，后续可平滑扩展为深度学习模型。结合大语言模型实现内容的智n能理解、自动摘要与规范管理，让每一段影像都清晰可见、有序可查。

---

### 二、核心目标

1. **一站式影音管家**：扫描、导入、分类、收藏、播放统计本地影音文件，构建个人多媒体资料库。
2. **多策略增强画面**：支持可扩展的增强方法插件，初期通过线性拉伸快速落地实时增强与离线处理能力，后续可按需集成 MAXIM 等深度学习模型。
3. **实时增强任意屏幕内容**：对播放器、网页、监控等画面进行低延迟透明覆盖增强，支持多种模式切换。
4. **离线内容修复**：对已保存的视频或图像文件进行增强处理，导出更清晰的版本并自动纳入管理库。
5. **AI 深度赋能**：利用云端大模型自动生成视频简介、语音转文字、字幕摘要、智能重命名、同类内容推荐等，降低人工整理成本。
6. **录制与分享**：集成录屏功能，可录制增强后的屏幕画面或原始画面，录制文件自动入库，形成“获取→增强→管理→分享”的闭环。

---

### 三、目标用户

- 有大量家庭录像、旅行视频需要整理和修复的普通用户。
- 需要在低光、雾霾天气下查看监控或行车记录仪画面的安防人员。
- 经常观看在线低质量视频并希望实时提升画质的影音爱好者。
- 盲人或视障用户群体，通过 AI 语音描述理解画面内容。

### 四、功能模块

#### 4.1 影音管理模块
- **文件扫描与导入**：支持指定文件夹递归扫描，识别常见视频、音频、图像格式，批量导入数据库。
- **元数据提取**：自动获取文件类型、时长、分辨率、文件大小等基础信息。
- **多维浏览与筛选**：可按类型（视频/音频/图片）、名称关键词、播放次数、收藏状态、AI 标签等过滤和排序。
- **收藏与标签**：支持一键收藏/取消收藏，用户可自定义标签。
- **播放记录**：自动记录每次播放的时间与进度，提供“最近播放”快速访问入口。
- **播放统计**：统计各媒体文件的播放次数、总时长，支持图表展示。

#### 4.2 图像增强模块（多方法可扩展，初期线性拉伸）
- **插件式增强架构**：增强模块设计为算法可替换的架构，每个增强方法实现统一接口，支持未来添加不同算法（如直方图均衡化、CLAHE、深度学习模型 MAXIM 等）。
- **初始方法：线性拉伸**  
  为快速开发和验证系统流程，初期选用简单高效的线性拉伸作为默认增强方法。  
  - 通过将输入图像像素值按线性映射扩展到全动态范围，有效改善低光、低对比度问题。  
  - 无复杂依赖，纯 CPU 即可实时处理，延迟极低。  
  - 提供参数可调的增强强度（拉伸范围、亮度偏移等），用户可实时拖动滑块调节效果。
- **方法切换与混合**：UI 中预留增强方法选择器，未来可无缝切换至其他算法或 AI 模型，也支持方法组合（如先用线性拉伸快速预览，再调用深度学习模型精细处理）。
- **实时增强与离线增强共享同一接口**：无论屏幕覆盖增强还是文件批量处理，均通过调用当前激活的增强方法实现，降低维护成本。

#### 4.3 实时屏幕增强模块
- **透明覆盖窗口**：以无边框透明窗口悬浮于屏幕最顶层，显示增强后的画面，可拖动、锁定位置、调节透明度。
- **多模式选择**：初期仅提供线性拉伸模式，未来可扩展至去雾、去雨、综合增强等。
- **全局热键**：使用组合键快速开启/关闭增强、切换方法、调节强度。
- **区域增强**：用户可框选屏幕上任意区域，仅增强该区域内容。
- **点击穿透**：增强窗口不阻挡鼠标操作，用户可继续正常使用下层应用。

#### 4.4 离线文件增强模块
- **入口**：在影音库内右键任意视频/图像，选择“AI 增强”，或通过菜单批量处理。
- **处理选项**：选择当前可用的增强方法（初期为线性拉伸），调整强度等参数。
- **预览对比**：处理完成后弹出原图/增强图滑条对比窗口，用户确认后导出增强文件。
- **自动入库**：增强后的新文件自动添加到影音库，并在数据库中关联源文件，形成版本记录。

#### 4.5 AI 智能模块
- **自动生成简介与标签**：对视频文件抽取关键帧，调用多模态大模型分析内容，生成文字描述和场景标签；也可基于音频转写文字生成摘要。
- **语音转文字**：提取视频音轨或单独上传音频，调用云端 Whisper API 将语音转为文字，存入数据库。
- **字幕摘要**：基于完整字幕文本，让大模型提炼剧情梗概或关键信息。
- **智能重命名**：根据 AI 分析的时间、地点、事件等，将文件名规范化为“日期_地点_事件_画质特征.扩展名”格式，支持批量处理。
- **智能推荐**：基于 AI 标签相似度或大模型分析播放历史，推荐库内同类影片，并给出推荐理由。

#### 4.6 录屏模块
- **录制源**：支持录制全屏、特定窗口或自定义区域。
- **增强录制**：可在录制时应用当前增强方法（初期为线性拉伸），直接保存增强后的画面。
- **音频支持**：同步录制系统声音、麦克风或两者混合。
- **编码与格式**：采用 GPU 硬件编码输出 MP4 文件，兼顾效率与质量。
- **录制控制**：开始/暂停/停止按钮、状态指示器、时长显示，支持全局热键。
- **录制后处理**：录制文件自动入库，支持立即预览、编辑或二次增强。

#### 4.7 数据统计与系统设置
- **使用统计**：个人面板展示累计增强次数、录制时长、文件数量等。
- **参数预设**：保存常用的增强参数组合为预设，一键调用。
- **API 配置**：可配置大模型 API 密钥、语音识别服务提供商等。
- **性能设置**：选择推理设备（CPU/GPU）、质量-速度平衡档位。

### 五、技术架构

#### 5.1 总体架构
系统采用 **C# WPF 桌面端 + Python 推理服务（可选）** 的分离式架构。初期增强方法（线性拉伸）纯 C# 实现，无需额外服务；未来扩展深度学习模型时，可通过本地 HTTP 或命名管道调用 Python 推理后端，最终打包为单一安装程序。  

主要层次：
- **表现层 (WPF)**：UI 界面、屏幕捕获、透明覆盖窗口、录屏控制。
- **业务逻辑层**：影音管理、增强任务调度、增强方法插件管理、外部 API 调用、文件处理流程。
- **增强方法层**：定义增强接口 `IEnhancementMethod`，线性拉伸为第一个实现，未来可添加 `HistogramEqualization`、`CLAHE`、`MAXIMMethod` 等。
- **数据访问层**：通过 Entity Framework Core 操作本地 SQLite 数据库。
- **AI 推理层（可选扩展）**：独立的 Python 进程加载深度学习模型，提供 RESTful 推理接口，未来可导出 ONNX 集成到 C# 端。
- **外部服务层**：接入 OpenAI Whisper API、GPT-4V 等多模态大模型。

#### 5.2 核心技术栈
- **桌面框架**：.NET 8 WPF，透明窗口与 GPU 渲染。
- **屏幕捕获**：SharpDX / DXGI Desktop Duplication API，GPU 零拷贝获取屏幕纹理。
- **初始增强方法**：C# 原生线性拉伸（像素值线性映射），无额外依赖，性能优异。
- **可扩展增强框架**：接口化设计，支持运行时切换或加载新方法。
- **视频/音频处理**：FFmpeg 用于音轨提取、编解码、帧合成；NAudio 用于系统音频捕获。
- **语音识别**：OpenAI Whisper API（纯云端）。
- **大语言模型**：OpenAI GPT-4V / Claude Vision，支持本地 Ollama 隐私部署。
- **数据库**：SQLite + EF Core。
- **编码加速**：NVENC / Intel QSV / AMD AMF 硬件编码。

#### 5.3 数据流
- **增强数据流（初始阶段）**：屏幕/文件 → 图像预处理（格式转换）→ 调用线性拉伸算法（C# 内）→ 结果直接显示/录制/编码。
- **增强数据流（未来深度学习）**：屏幕/文件 → 预处理 → HTTP 请求 → Python 推理 → 增强结果返回 → 显示/录制/编码。
- **影音管理数据流**：用户操作 → WPF ViewModel → 业务服务 → EF Core → SQLite。
- **AI 智能数据流**：文件 → 抽取关键帧/音轨 → 上传至大模型 API → 分析结果解析 → 存入数据库 → UI 更新。

### 六、数据库设计概要

使用 SQLite 数据库，主要表包括：
- **MediaFiles**：影音文件元数据（名称、类型、时长、路径、AI标签、摘要、收藏状态等）。
- **PlayHistory**：播放记录（关联媒体ID、播放时间、进度）。
- **EnhancementLog**：增强历史（关联源文件、增强方法名称、参数、输出路径、时间）。
- **Recordings**：录屏文件记录（路径、时长、是否增强、音频来源等）。
- **Presets**：增强参数预设（方法名、参数JSON）。
- 可选扩展表：用户偏好、集合管理、AI检测事件等。

### 七、项目特色与创新点

- **渐进式增强框架**：初期用最简线性拉伸快速上线核心流程，架构预留扩展点，未来可无缝接入深度学习模型，兼顾开发速度与长期演进。
- **管理 + 增强融合**：在文件管理界面直接提供画质修复入口，增强文件自动归档，形成统一资产库。
- **全链路 AI 智能化**：语音转文字、字幕摘要、智能重命名、推荐等功能，将影音整理工作语义化、自动化。
- **实时与离线互补**：既可在任意窗口上直接覆盖增强，又可对本地文件深度修复并导出。
- **增强录屏一体化**：录制时可叠加增强效果，适用于保存在线低质量视频或制作教学素材。
- **无障碍辅助潜力**：结合大模型生成画面文字描述，帮助视觉障碍用户理解影像内容。

### 八、开发计划建议

#### 阶段一：基础框架与影音管理
- 搭建 WPF 主界面。
- 实现文件扫描、导入、SQLite 数据库读写。
- 完成影音列表、筛选、播放统计和收藏功能。

#### 阶段二：线性拉伸增强集成
- 设计增强方法接口，实现线性拉伸算法（C#）。
- 集成实时屏幕增强：透明覆盖窗口、热键控制、强度调节。
- 实现离线文件增强流程（单张图像，视频逐帧处理）与预览对比。

#### 阶段三：AI 智能模块
- 集成 Whisper API 实现语音转文字。
- 接入多模态大模型，完成自动摘要、标签生成。
- 开发智能重命名与基于标签的推荐功能。

#### 阶段四：录屏与系统完善
- 实现录屏模块（含增强录制、硬件编码）。
- 增加全局热键、系统托盘、设置面板。
- 性能优化、打包测试、撰写文档。

#### 阶段五：增强方法扩展（未来）
- 添加更多传统增强算法（直方图均衡化、CLAHE）。
- 引入深度学习模型（如 MAXIM），支持 Python 推理服务调用。
- 实现方法切换与混合增强。

### 九、预期成果

一个功能完整、易于扩展的 Windows 桌面应用，用户可以通过它：
- 方便地管理本地所有影音文件。
- 使用简单的线性拉伸实时提升任何屏幕区域的暗部与对比度。
- 一键修复画质低下的旧视频，并自动整理入库。
- 让 AI 自动为大批量文件生成规范名称和内容简介。
- 录制网络课程、监控画面时同步增强，获得更清晰的存档。

该项目不仅能满足课程作业的各项要求，同时其开放式增强架构使其具备继续演进为专业视觉增强工具的潜力。
