# 影音智增强系统 — 完整类参考文档

---

## 一、Core 核心层

### 1.1 接口体系

Core 层定义了四个接口，构成增强方法的分层契约体系。

#### 继承关系

```
IEnhancementMethod          （根接口：名称、描述、参数列表）
  ├── IRealTimeEnhancer      （实时逐帧增强：SupportsRealTime + Enhance(byte[])）
  ├── INativeEnhancement     （离线同步增强：Enhance(BitmapSource)）
  └── IOnnxEnhancement       （ONNX 深度学习增强：ModelFileName + EnhanceAsync）
```

#### IEnhancementMethod（根接口）

**文件**：`Core/IEnhancementMethod.cs`

所有增强方法的共同基接口，仅定义三个只读属性的访问器，无方法。

| 成员 | 签名 | 类型 | 说明 |
|------|------|------|------|
| 属性 | `string Name { get; }` | `string` | 增强方法名称，如 "线性拉伸" |
| 属性 | `string Description { get; }` | `string` | 方法描述文本 |
| 属性 | `List<EnhancementParameter> Parameters { get; }` | `List<EnhancementParameter>` | 可调参数列表，供 UI 滑块绑定与算法参数传递 |

**实现者**：`LinearStretchMethod`（间接通过 `INativeEnhancement` / `IRealTimeEnhancer`）、`MultinexNanoMethod`（间接通过双接口）

**消费者**：`EnhancementRegistry.GetMethod()` 返回值为 `IEnhancementMethod?`

---

#### IRealTimeEnhancer（实时增强接口）

**文件**：`Core/IRealTimeEnhancer.cs`，继承 `IEnhancementMethod`

定义字节数组级逐帧屏幕增强契约，消费者仅依赖此接口，不依赖具体算法实现。

| 成员 | 签名 | 类型 | 说明 |
|------|------|------|------|
| 属性 | `bool SupportsRealTime { get; }` | `bool` | 是否支持实时处理（延迟 < 33ms 可达 30FPS）。为 false 的方法不出现在实时增强列表中 |
| 方法 | `byte[] Enhance(byte[] pixels, int width, int height, int stride, IReadOnlyDictionary<string, double>? parameters = null)` | `byte[]` | 对 BGRA32 原始像素字节数组执行增强，返回同格式同尺寸的结果数组 |

**方法参数说明**：

| 参数 | 类型 | 含义 |
|------|------|------|
| `pixels` | `byte[]` | BGRA32 格式的原始像素字节数组 |
| `width` | `int` | 图像宽度（像素） |
| `height` | `int` | 图像高度（像素） |
| `stride` | `int` | 每行字节数（行跨度），通常 = width * 4 |
| `parameters` | `IReadOnlyDictionary<string, double>?` | 可选参数字典（键 = 参数 Key，值 = 当前设置）；为 null 时使用默认值 |

**实现者**：`LinearStretchMethod`、`MultinexNanoMethod`

**消费者**：
- `FullscreenEnhanceWindow` — 全屏增强覆盖窗口，每帧捕获屏幕后调用 `Enhance()` 再渲染
- `VideoEnhancer` — 视频逐帧增强，通过 `Task.Run` 在线程池中调用
- `ScreenRecorder` — 录制时可选增强，在 `EnhanceFrames` 中作为回退方案调用

#### IOnnxEnhancement（ONNX 深度学习增强接口）

**文件**：`Core/IOnnxEnhancement.cs`，继承 `IEnhancementMethod`

为基于 ONNX 深度学习模型的图像增强提供契约。

| 成员 | 签名 | 类型 | 说明 |
|------|------|------|------|
| 属性 | `string ModelFileName { get; }` | `string` | 模型文件名（不含路径），模型文件位于运行目录的 `OnnxModels/` 子目录下 |
| 属性 | `int MaxInferenceSize { get; }` | `int` | 推理尺寸上限（长边像素）。超过此值的输入先缩放到此尺寸再推理，之后放大回原分辨率。0 表示不限制。默认 480 适合实时场景 |
| 方法 | `Task<BitmapSource> EnhanceAsync(BitmapSource input)` | `Task<BitmapSource>` | 异步执行 ONNX 推理增强，使用内部缓存的模型 |
| 方法 | `Task<BitmapSource> EnhanceAsync(BitmapSource input, string modelPath)` | `Task<BitmapSource>` | 异步执行 ONNX 推理增强，从外部指定模型文件路径加载（适合测试不同权重文件） |

**实现者**：`MultinexNanoMethod`

**消费者**：
- `MainViewModel` 离线增强预览（`RefreshOfflinePreviewAsync`）
- `MainViewModel.EnhanceFile` 命令中的离线图片增强路径
- `ScreenRecorder.EnhanceFrames` 作为后处理阶段优先使用的增强方式

### 1.2 EnhancementParameter — 增强参数模型

**文件**：`Core/EnhancementParameter.cs`

一个简单的 POCO 数据模型类，无继承、无接口实现、无方法、无字段。仅提供 6 个自动属性描述增强算法中单个参数的元数据。

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `Name` | `string` | 参数显示名称（如 "对比度强度"） |
| `Key` | `string` | 参数键名，用于传递到算法实现（如 "contrast"） |
| `Value` | `double` | 当前值 |
| `Min` | `double` | 最小值 |
| `Max` | `double` | 最大值 |
| `Step` | `double` | 滑块步进 |

所有属性均为编译器自动生成 `{ get; set; }` 实现，无自定义逻辑。服务于 UI 滑块绑定和算法参数传递场景。

### 1.3 EnhancementRegistry — 增强方法注册中心

**文件**：`Core/EnhancementRegistry.cs`

轻量级注册中心 / 简单服务定位器。职责单一：集中管理所有 `IRealTimeEnhancer` 和 `IOnnxEnhancement` 实例的注册、查询和当前方法切换。

#### 字段

| 名称 | 类型 | 访问性 | 用途 |
|------|------|--------|------|
| `_methods` | `Dictionary<string, IRealTimeEnhancer>` | private readonly | 存储已注册的实时增强方法，以名称（Name）为键 |
| `_offlineMethods` | `Dictionary<string, IOnnxEnhancement>` | private readonly | 存储已注册的 ONNX 离线增强方法，以名称（Name）为键 |
| `_current` | `IRealTimeEnhancer?` | private | 当前选中的实时增强方法实例，未设置时为 null |

#### 属性

| 属性名 | 类型 | 说明 |
|--------|------|------|
| `Current` | `IRealTimeEnhancer?` | 返回 `_current`。如果从未设置过则返回 null |
| `CurrentMethodName` | `string` | 返回 `_current?.Name`，若为 null 则返回 "无" |
| `MethodNames` | `IReadOnlyList<string>` | `_methods.Keys` 与 `_offlineMethods.Keys` 拼接后的列表 |
| `Count` | `int` | 已注册方法总数（`_methods.Count + _offlineMethods.Count`） |
| `OfflineMethodNames` | `IReadOnlyList<string>` | 仅离线方法名称 |
| `RealTimeMethodNames` | `IReadOnlyList<string>` | 仅实时方法名称 |

#### 方法

| 方法签名 | 返回类型 | 实现方式概述 |
|----------|----------|--------------|
| `Register(IRealTimeEnhancer method)` | `void` | 以 `method.Name` 为键写入 `_methods` 字典（同名覆盖）。若 `_current` 为 null（首次注册），则自动将 method 赋值给 `_current`（`??=` 运算符） |
| `RegisterOffline(IOnnxEnhancement method)` | `void` | 以 `method.Name` 为键写入 `_offlineMethods` 字典（同名覆盖）。不自动设为当前方法 |
| `SetCurrent(string methodName)` | `bool` | 在 `_methods` 字典中按名称查找，找到则更新 `_current` 并返回 true，否则返回 false。仅限实时方法 |
| `GetMethod(string methodName)` | `IEnhancementMethod?` | 先在 `_methods` 中查找，找到则返回 `IRealTimeEnhancer`；未找到则在 `_offlineMethods` 中查找；均未找到返回 null |
| `GetOfflineMethod(string methodName)` | `IOnnxEnhancement?` | 在 `_offlineMethods` 中按名称查找，找到返回实例，未找到返回 null |

### 1.4 LinearStretchMethod — 线性拉伸增强

**文件**：`Core/LinearStretchMethod.cs`

同时实现 `INativeEnhancement` 和 `IRealTimeEnhancer` 接口。实现基于直方图裁剪的线性拉伸（两遍扫描算法）。

#### 属性

| 属性名 | 类型 | 来源接口 | 说明 |
|--------|------|----------|------|
| `Name` | `string` | IEnhancementMethod | 固定返回 "线性拉伸" |
| `Description` | `string` | IEnhancementMethod | 返回 "将像素值线性映射到全动态范围，改善低光照和低对比度" |
| `SupportsRealTime` | `bool` | IRealTimeEnhancer | 返回常量 `true` |
| `Parameters` | `List<EnhancementParameter>` | IEnhancementMethod | 含 2 个元素：`contrast`（0.5-2.0，步进 0.1，默认 1.0）和 `brightness`（-50-50，步进 1，默认 0） |
| `Contrast` | `double` | 自定义便捷包装器 | get 返回 `Parameters[0].Value`；set 钳位到 `[0.5, 2.0]` 后写回 |
| `Brightness` | `double` | 自定义便捷包装器 | get 返回 `Parameters[1].Value`；set 钳位到 `[-50, 50]` 后写回 |

#### 方法

| 方法签名 | 返回类型 | 实现方式概述 |
|----------|----------|--------------|
| `Enhance(byte[] pixels, int width, int height, int stride, IReadOnlyDictionary<string, double>? parameters = null)` | `byte[]` | **byte[] 路径（实时）**。两遍扫描：第一遍建 ITU-R BT.601 亮度直方图并裁剪首尾各 2% 像素确定 minVal/maxVal；第二遍通过 `Buffer.BlockCopy` 复制原数组后逐像素应用线性拉伸公式 `(pixel - minVal) * (255.0 / range) * contrast + brightness`，钳位至 `[0, 255]`，四舍五入后返回新数组 |
| `Enhance(BitmapSource input)` | `BitmapSource` | **BitmapSource 路径（离线）**。构造 `WriteableBitmap` → `Lock()` 锁住缓冲区 → `unsafe` 指针遍历 → 第一遍找全局极值 → 第二遍逐通道写回拉伸结果 → `AddDirtyRect` + `Unlock()`。若格式非 Bgra32 则包装为 `FormatConvertedBitmap` 返回 |

#### 两遍扫描算法详细步骤

1. **第一遍 - 直方图统计**：分配 `int[256]` 直方图数组，遍历 BGRA32 像素，使用 ITU-R BT.601 luma 公式 `(R*299 + G*587 + B*114) / 1000` 计算亮度值（0-255），累加各亮度级像素计数。

2. **直方图裁剪（确定范围）**：裁剪首尾各 2% 像素（`totalPixels * 0.02`）。从暗端（0）向亮端（255）扫描，累计像素数首次超过裁剪量时确定为 `minVal`；从亮端向暗端扫描确定 `maxVal`。若 `maxVal <= minVal`（均匀图像），则设为 `[0, 255]`。

3. **第二遍 - 线性映射**：遍历每个像素的 B/G/R 三个通道（Alpha 保持不变），应用公式 `(pixel - minVal) * (255.0 / range) * contrast + brightness`，钳位至 `[0, 255]`，使用 `(byte)(pixel + 0.5)` 四舍五入（非银行家舍入）。

#### byte[] 路径和 BitmapSource 路径的区别

| 维度 | byte[] 路径 | BitmapSource 路径 |
|------|-------------|-------------------|
| 消费者场景 | 全屏实时增强、视频逐帧增强 | 离线图片增强预览、文件增强 |
| 直方图亮度计算 | ITU-R BT.601 加权亮度 | 逐通道独立跟踪全局极值 |
| 内存策略 | `Buffer.BlockCopy` 减少分配 | `unsafe` 指针直接写回 `WriteableBitmap` |
| 参数来源 | 通过参数字典运行时覆盖 | 读取 `Parameters` 列表（UI 滑块绑定值） |

### 1.5 MultinexNanoMethod — ONNX 深度学习增强

**文件**：`Core/MultinexNanoMethod.cs`

同时实现 `IRealTimeEnhancer` 和 `IOnnxEnhancement` 双接口。15K 参数的 Retinex ONNX 模型，在 CPU 上约 30ms/帧。

#### 属性

| 属性名 | 类型 | 来源接口 | 说明 |
|--------|------|----------|------|
| `Name` | `string` | IEnhancementMethod | 返回 "Multinex Nano 低光增强" |
| `Description` | `string` | IEnhancementMethod | 返回 "超轻量 Retinex 增强（15K 参数），适合实时场景" |
| `SupportsRealTime` | `bool` | IRealTimeEnhancer | 返回 `true`，CPU 上约 30ms/帧 |
| `Parameters` | `List<EnhancementParameter>` | IEnhancementMethod | 返回空列表（效果完全由 ONNX 模型权重决定，无用户可调参数） |
| `ModelFileName` | `string` | IOnnxEnhancement | 返回 `"multinex_nano_lolv2_syn.onnx"` |
| `MaxInferenceSize` | `int` | IOnnxEnhancement | 返回 `0`，全分辨率推理，不降采样 |

#### 字段

| 名称 | 类型 | 访问性 | 用途 |
|------|------|--------|------|
| `_session` | `InferenceSession?` | private | 缓存的 ONNX 推理会话，延迟初始化，首次调用时才加载 |
| `_lock` | `object` (readonly) | private | 双重检查锁定的同步对象，确保 Session 的创建与释放线程安全 |

#### 方法

| 方法签名 | 返回类型 | 实现方式概述 |
|----------|----------|--------------|
| `Enhance(byte[] pixels, int width, int height, int stride, IReadOnlyDictionary<string, double>? parameters = null)` | `byte[]` | **IRealTimeEnhancer 实现**。调用 `EnsureSession()` 确保 Session 就绪 → `OnnxModelHelper.Preprocess`（BGRA32 → NCHW float32，maxSize=0 全分辨率）→ `OnnxModelHelper.RunInference` → `OnnxModelHelper.Postprocess`（NCHW → BGRA32 bytes）。专为全屏/视频逐帧增强设计 |
| `EnhanceAsync(BitmapSource input)` | `Task<BitmapSource>` | **IOnnxEnhancement 实现（缓存 Session）**。`EnsureSession()` 确保就绪 → UI 线程上 `input.CopyPixels` 读取像素 → `Task.Run` 中执行 Preprocess/RunInference/Postprocess。Session 被缓存复用 |
| `EnhanceAsync(BitmapSource input, string modelPath)` | `Task<BitmapSource>` | **IOnnxEnhancement 实现（外部模型）**。整个流程放在 `Task.Run` 中异步执行。每次调用新建独立的 `InferenceSession`（`using var session = new InferenceSession(modelPath)`），用后即弃。适合测试不同权重文件 |
| `EnsureSession()` | `void` | **双重检查锁定**。快速路径：若 `_session != null` 直接返回；否则进入 `lock(_lock)` 再检查一次，若仍为 null 则从 `{BaseDirectory}/OnnxModels/{ModelFileName}` 路径加载 `new InferenceSession(...)` |
| `Dispose()` | `void` | 在 `lock(_lock)` 保护下调用 `_session?.Dispose()` 并将 `_session` 置为 null |

#### 双接口实现的用途分工

| 接口 | 用途 | 调用方 |
|------|------|--------|
| `IRealTimeEnhancer` | 全屏实时增强（byte[] 路径，30fps）、视频逐帧增强 | `FullscreenEnhanceWindow`、`VideoEnhancer`、`ScreenRecorder`（回退） |
| `IOnnxEnhancement` | 离线图片增强预览（BitmapSource 路径、高质量）、文件增强 | `MainViewModel` 离线增强面板、`ScreenRecorder.EnhanceFrames`（后处理优先） |

#### 延迟加载 + 双重检查锁定的原理

1. 首次调用 `Enhance` 或 `EnhanceAsync` 时，通过 `EnsureSession()` 触发模型加载。
2. **快速路径**：检查 `_session != null`，若已加载则直接返回（无锁开销）。
3. **慢速路径**：`_session == null` 时进入 `lock(_lock)` 块。
4. **双重检查**：lock 内部再次检查 `_session == null`，防止多个线程同时通过快速路径后争抢创建。
5. 仅第一个进入 lock 的线程从磁盘加载 ONNX 模型文件并创建 `InferenceSession`，其余线程复用。
6. `Dispose()` 同样在 lock 保护下进行，避免与 `EnsureSession` 并发冲突。

### 1.6 OnnxModelHelper — ONNX 预处理/后处理

**文件**：`Core/OnnxModelHelper.cs`

静态工具类，核心职责：在 WPF 图像格式和 ONNX 模型格式之间做双向转换。WPF 侧：`BitmapSource` / BGRA32 `byte[]`；ONNX 侧：NCHW float32 `[0.0, 1.0]`。

#### 方法

| 方法签名 | 返回类型 | 实现方式概述 |
|----------|----------|--------------|
| `GetInferenceSize(int width, int height, int maxSize)` | `(int w, int h)` | 根据原始尺寸和 maxSize 计算推理目标尺寸，保持宽高比。若 maxSize <= 0 或输入尺寸已在限制内，返回原始尺寸；否则以长边为基准等比缩放 |
| `ResizeBilinear(byte[] src, int srcW, int srcH, int srcStride, int dstW, int dstH)` | `byte[]` | 对 BGRA32 像素数组做双线性插值缩放。为每个目标像素找到源图中 4 个邻居像素（左上/右上/左下/右下），按水平和垂直方向的小数部分作为权重混合。对 B/G/R/A 四个通道分别做插值，结果四舍五入后钳位到 [0,255]。边界像素使用 Clamp to edge |
| `Preprocess(byte[] pixels, int width, int height, int stride, int maxSize)` | `(float[] data, int infW, int infH, int origW, int origH, bool downscaled)` | **byte[] 路径预处理**。先调 `GetInferenceSize` 计算推理尺寸，需要时调用 `ResizeBilinear` 缩放。然后将 BGRA32 byte[] 转换为 NCHW float32 格式：`result[C*H*W + Y*W + X]`，C=0:R（BGRA[+2]），C=1:G（BGRA[+1]），C=2:B（BGRA[+0]），值域除以 255 归一化到 [0.0, 1.0] |
| `Postprocess(float[] data, int infH, int infW, int origW, int origH, int origStride, bool downscaled)` | `byte[]` | **byte[] 路径后处理**。若未降采样，直接调用 `PostprocessDirect` 转换；否则先在推理尺寸下生成 BGRA byte[]，再通过 `ResizeBilinear` 放大回原始分辨率 |
| `PostprocessDirect(float[] data, int height, int width, int stride)` | `byte[]` | **直出版本（无缩放）**。将 NCHW float32 逆映射为 BGRA32 byte[]：C=0(R) → BGRA[+2]，C=1(G) → BGRA[+1]，C=2(B) → BGRA[+0]。值乘以 255 后通过 `ClampToByte` 钳位到 [0,255]。Alpha 固定为 255 |
| `Preprocess(BitmapSource source)` | `float[]` | **BitmapSource 路径预处理**。统一转为 BGRA32 格式（`FormatConvertedBitmap`）→ `CopyPixels` 获取 byte[] → 遍历转换 BGRA 字节序为 NCHW float32 [0,1] RGB |
| `Postprocess(float[] data, int height, int width)` | `BitmapSource` | **BitmapSource 路径后处理**。将 NCHW float32 转换为 BGRA32 byte[] → `BitmapSource.Create` 创建位图（DPI=96）→ `Freeze()` 使其不可变/线程安全 |
| `RunInference(InferenceSession session, float[] input, int height, int width)` | `float[]` | 将 NCHW float32 数据构造为形状 `[1, 3, height, width]` 的 `DenseTensor<float>` → 包装为名为 "input" 的 `NamedOnnxValue` → `session.Run()` → 取返回集合中第一个输出 tensor 转为 float[] |
| `ClampToByte(float v)` | `byte` | 加 0.5f 四舍五入取整 → `Math.Clamp` 限制到 [0, 255] → 转为 byte |

#### BGRA32 与 NCHW float32 的内存布局变换公式

```
// 前向：BGRA32 byte[] → NCHW float32[]
// 对于位置 (C, Y, X):
//   C=0(R): 取 pixels[ Y*stride + X*4 + 2 ] / 255.0f
//   C=1(G): 取 pixels[ Y*stride + X*4 + 1 ] / 255.0f
//   C=2(B): 取 pixels[ Y*stride + X*4 + 0 ] / 255.0f
// 存储位置: result[ C*H*W + Y*W + X ]

// 逆向：NCHW float32[] → BGRA32 byte[]
//   BGRA[+0](B): data[ 2*H*W + Y*W + X ] * 255 → byte
//   BGRA[+1](G): data[ 1*H*W + Y*W + X ] * 255 → byte
//   BGRA[+2](R): data[ 0*H*W + Y*W + X ] * 255 → byte
//   BGRA[+3](A): 255（固定不透明）
```

#### Bilinear 双线性插值算法步骤

1. 计算源图到目标图的缩放比：`scaleX = srcW / dstW`，`scaleY = srcH / dstH`。
2. 对每个目标像素 `(dx, dy)`，计算其在源图中的浮点坐标：`sx = dx * scaleX`，`sy = dy * scaleY`。
3. 确定 4 个邻居整数坐标：`x0 = floor(sx)`，`x1 = min(x0+1, srcW-1)`，`y0 = floor(sy)`，`y1 = min(y0+1, srcH-1)`。
4. 计算小数部分权重：`fx = sx - x0`，`fy = sy - y0`。
5. 对 B/G/R/A 四个通道分别做双线性插值：`result = (1-fy)*((1-fx)*p00 + fx*p10) + fy*((1-fx)*p01 + fx*p11)`。
6. 结果四舍五入取整并钳位到 [0, 255]。

### 4.5 DxgiScreenCapture — DXGI 桌面捕获

**文件**：`Services/DxgiScreenCapture.cs`，实现 `IDisposable`

无构造函数依赖注入（无参构造函数，依赖通过 `Initialize()` 延迟创建）。

**属性**：

| 属性 | 类型 | 说明 |
|------|------|------|
| `IsReady` | `bool` (get-only) | 返回 `_initialized`，指示捕获是否已就绪 |
| `Width` | `int` (get-only) | 屏幕宽度（物理像素） |
| `Height` | `int` (get-only) | 屏幕高度（物理像素） |

**方法**：

| 方法签名 | 返回类型 | 实现方式概述 |
|----------|----------|--------------|
| `Initialize()` | `bool` | 顺序执行 8 步：创建 DXGI Factory → 枚举适配器 → 枚举输出并获取屏幕尺寸 → 创建 D3D11 设备 → 调用 `DuplicateOutput` 创建桌面复制对象 → 创建 CPU-staging 纹理（`B8G8R8A8_UNORM`，`CPUAccessFlags=0x20000`）。任一步骤失败则立即 `Cleanup()` 返回 false |
| `CaptureFrame(int maxWaitMs = 100)` | `byte[]?` | 1. 循环调用 `AcquireNextFrame` 获取新帧，每次等待最多 16ms，累计不超过 `maxWaitMs`。收到 `DXGI_ERROR_ACCESS_LOST` 则置 `_initialized = false` 并返回 null → 2. `CopyResource` 将 GPU 帧纹理拷贝到 staging 纹理 → 3. `Flush` 刷新 GPU 命令队列 → 4. `ReleaseFrame` 归还帧 → 5. `Map` 映射 staging 纹理到 CPU 内存 → 6. `Marshal.Copy` 逐行拷贝像素数据到 byte[]（处理 RowPitch 与目标 Stride 差异）→ 7. `finally` 中 `Unmap` 解除映射 |
| `Dispose()` | `void` | 调用 `Cleanup()` 释放所有 COM 资源 → `GC.SuppressFinalize(this)` |

该类的核心在于通过 P/Invoke 调用 `d3d11.dll` 和 `dxgi.dll` 的 COM 接口，使用 vtable 直接调用（`GetVtblFunc<T>`）实现 DXGI Desktop Duplication API，实现 GPU 零拷贝屏幕捕获。

### 6.1 FullscreenEnhanceWindow — 全屏增强覆盖窗口

**文件**：`Views/FullscreenEnhanceWindow.xaml.cs`，基类：`Window`

无 ViewModel 依赖。直接持有 `IRealTimeEnhancer` 引用并自驱动。

#### 关键字段

| 字段 | 类型 | 用途 |
|------|------|------|
| `_method` | `IRealTimeEnhancer?` | 增强算法实现 |
| `_params` | `IReadOnlyDictionary<string, double>?` | 算法调优参数 |
| `_dxgiCapture` | `DxgiScreenCapture?` | DXGI Desktop Duplication API 封装 |
| `_useDxgi` | `bool` | 是否使用 DXGI（false → GDI 回退） |

#### Win32 关键常量

- `WS_EX_TRANSPARENT`（0x00000020）：鼠标点击穿透，保证覆盖全屏时不影响底层窗口操作
- `WDA_EXCLUDEFROMCAPTURE`（0x00000011）：排除自身递归捕获（消除画中画效应）
- `HOTKEY_ID=9001`：注册 F11 全局热键，保证全屏覆盖时用户仍能退出

#### 方法

| 方法 | 实现方式概述 |
|------|--------------|
| `Start(IRealTimeEnhancer, IReadOnlyDictionary<string, double>?)` | 存储方法/参数 → 计算 DPI → 初始化 DXGI（失败回退 GDI）→ `Show()` 窗口 → 设置鼠标穿透 → 启动 `CaptureLoop` |
| `Stop()` | 取消 `_cts` → 注销 F11 热键 → `Hide()` → 触发 `Stopped` 事件 |
| `CaptureLoop(CancellationToken)` | 循环：DXGI 模式 30fps（33ms 间隔）、GDI 模式 10fps（100ms 间隔） |
| `CaptureOneFrame(CancellationToken)` | 后台线程捕获 → `Enhance()` 增强 → `BitmapSource.Create` + `Freeze()` → UI 线程 `Dispatcher.Invoke` 更新 `Display.Source` |

#### 窗口生命周期

```
构造 → SourceInitialized（注册热键/穿透）→ Start()（启动循环）
  └── 运行（30fps DXGI / 10fps GDI）
      └── F11 热键 → Stop() → Stopped 事件 → 外部关闭
```

### 7.6 屏幕录制调用链

```
用户点击"开始录制"
  → MainViewModel.StartRecordingCommand
  ├── 计算物理屏幕分辨率 (VisualTreeHelper.GetDpi)
  ├── 选择增强器:
  │   ├── 离线方法 → GetSelectedOfflineMethod() → IOnnxEnhancement
  │   └── 实时方法 → IRealTimeEnhancer (兜底 LinearStretch)
  ├── new ScreenRecorder(w, h, outputDir, enhancer, offlineEnhancer, params, fps)
  │   └── Start()
  │       └── RecordingLoop:
  │           ├── DxgiScreenCapture.Initialize() → CaptureFrame()
  │           └── GDI Graphics.CopyFromScreen (回退)
  │           └── SaveAsJpeg (GDI Bitmap.LockBits + Marshal.Copy)
  └── UpdateRecordingDurationAsync (500ms 轮询更新 UI)

用户点击"停止录制"
  → MainViewModel.StopRecordingCommand
  └── _recorder.StopAsync(status)
      ├── 取消录制循环
      ├── 等待录制 Task 结束
      ├── 有增强器 → EnhanceFrames (后台 Task)
      │   ├── 优先: _offlineEnhancer.EnhanceAsync(BitmapSource)
      │   └── 回退: _enhancer.Enhance(byte[])
      └── TryFfmpegAsync → 五级编码器降级链 → MP4 输出
      └── DataService.AddMediaFileAsync → 入库
      └── DataService.AddRecordingAsync → 录制记录
```

### 7.7 AI 对话调用链

```
用户输入消息 → MainViewModel.SendAiMessageCommand
  ├── GetSelectedAiFiles() → 取 IsSelected 的 MediaFile
  ├── 构建 ChatMessage (Role="user") → 添加到 AiMessages
  ├── 添加 thinking 占位消息
  ├── AiService.ChatAsync(history, selectedFiles, systemPrompt)
  │   ├── API 已配置:
  │   │   └── BuildMessagesAsync → 组装多模态消息
  │   │       ├── system prompt
  │   │       ├── 最近 10 条历史
  │   │       └── BuildFileContextAsync → 图片 base64 / 视频关键帧 base64
  │   │           └── ExtractVideoFrames → ffmpeg 提取关键帧（临时文件立即删除）
  │   │   └── HttpClient.PostAsync → 阿里百炼 DashScope API
  │   │       └── 解析 choices[0].message.content
  │   └── API 未配置 / 异常:
  │       └── FallbackChat → 基于 systemPrompt 关键词分发到离线模板
  │           ├── "简介" → TemplateDescription (文件元数据)
  │           ├── "增强" → TemplateEnhancement
  │           ├── "美化" → TemplateBeautify
  │           ├── "数据" → TemplateDataSummary
  │           └── 其他 → TemplateGeneral (功能介绍 / 通用引导)
  └── 移除 thinking 消息 → 添加 assistant 回复到 AiMessages

AI 图像编辑:
  用户选择图片 + 输入提示词 → MainViewModel.AiEditGenerateCommand
  ├── 校验 ChatApiEndpoint + ChatApiKey 已配置
  ├── 读取 AiEditOriginal → JpegBitmapEncoder → Base64
  ├── AiService.GenerateImageAsync(prompt, imageBase64)
  │   ├── ResolveFormat() → dashscope / openai 格式
  │   └── PostImageAsync → 构造 JSON body → POST → 解析响应 URL → 下载图片字节
  └── 加载结果到 AiEditResult → UI 显示
```
