# 录屏与实时增强技术总结

## 概述

本项目实现了两种屏幕捕获与视频处理功能：

1. **实时全屏增强** — 捕获屏幕内容 → 增强 → 全屏覆盖显示，不干扰用户操作
2. **录屏** — 捕获屏幕帧 → 可选增强 → 编码为 MP4 视频

两者共享同一套帧捕获管线，区别在于输出方式不同。

---

## 一、帧捕获管线

### 1.1 DXGI Desktop Duplication（首选）

```csharp
DxgiScreenCapture.cs
```

使用纯 P/Invoke 调用 `d3d11.dll` 和 `dxgi.dll`，零外部依赖。

**流程**：
```
CreateDXGIFactory1 → EnumAdapters1 → EnumOutputs
  → D3D11CreateDevice → IDXGIOutput1.DuplicateOutput
  → 循环: AcquireNextFrame → CopyResource(Staging) → Unmap → ReleaseFrame
```

**关键实现**：
- 所有 COM 接口通过手动 VTable 调用（`Marshal.GetDelegateForFunctionPointer`）
- 创建 `ID3D11Texture2D` 作为 Staging 纹理（CPU 可读，`ResourceUsage.Staging`）
- 每帧通过 `CopyResource` 将 GPU 纹理拷贝到 Staging 纹理
- `Map` + `Marshal.Copy` 读取像素到 `byte[]`

**线程安全**：`D3D11` 设备与上下文有线程亲缘性——在初始化线程上使用。

### 1.2 GDI CopyFromScreen（回退）

```csharp
System.Drawing.Graphics.CopyFromScreen()
```

当 DXGI 初始化失败时（虚拟机、远程桌面等场景），自动回退到 GDI。

**流程**：
```
new Bitmap(w, h, Format32bppArgb)
  → Graphics.FromImage() → CopyFromScreen()
  → LockBits() → Marshal.Copy() → byte[]
```

**性能**：约 10 FPS，CPU 占用较高（~10%）。

## 二、全屏覆盖窗口（实时增强）

### 2.1 窗口创建

```csharp
FullscreenEnhanceWindow.xaml / .cs
```

| 属性 | 值 | 作用 |
|------|----|------|
| `WindowStyle` | `None` | 无边框 |
| `AllowsTransparency` | `True` | 创建分层窗口（Layered Window） |
| `Background` | `#01000000` | 几乎完全透明（alpha=1） |
| `Topmost` | `True` | 始终置顶 |
| `WindowState` | `Normal` | 手动铺满屏幕（不用 `Maximized`，避免与透明度冲突） |
| `ShowActivated` | `False` | 显示时不激活，不抢焦点 |

### 2.2 鼠标穿透（关键难点）

使用**分层窗口**（`AllowsTransparency=True`）+ **Win32 WS_EX_TRANSPARENT** 样式。

#### 为什么 `IsHitTestVisible=False` 不行？

WPF 的 `IsHitTestVisible` 仅阻止本窗口处理鼠标事件，但事件已进入窗口，**不会转发到下层窗口**，导致点击被吞掉。

#### 正确做法

```csharp
// 窗口显示后，通过 Win32 设置 WS_EX_TRANSPARENT
SetWindowLong(hwnd, GWL_EXSTYLE, exStyle | WS_EX_TRANSPARENT);
SetWindowPos(hwnd, ... SWP_FRAMECHANGED);
```

`WS_EX_TRANSPARENT` 让 Windows OS 将该窗口标记为"透明"，鼠标消息直接穿透到下层窗口。

#### 为什么不直接用 WS_EX_TRANSPARENT + 普通窗口？

普通窗口 + `WS_EX_TRANSPARENT` 在 `Topmost` 层时，Windows 找不到"下层窗口"（Topmost 层只有它自己），鼠标事件被丢弃。而**分层窗口**（Layered Window）的输入路径不同，`WS_EX_TRANSPARENT` 能正确穿透到非 Topmost 层。

### 2.3 递归消除

#### 问题

全屏窗口截屏时，窗口自身也会被 `CopyFromScreen` 捕获，导致"画中画"递归效应。

#### 方案：`SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`

```csharp
[DllImport("user32.dll")]
static extern bool SetWindowDisplayAffinity(IntPtr hwnd, uint dwAffinity);

SetWindowDisplayAffinity(hwnd, 0x11); // WDA_EXCLUDEFROMCAPTURE
```

该 API 告知 DWM（Desktop Window Manager）本窗口**不应出现在截屏中**。`CopyFromScreen` 捕获的图像中将不包含本窗口，彻底消除递归。

> **要求**：Windows 10 2004+（本应用仅支持 Windows 10/11）

### 2.4 捕获循环

```csharp
private async Task CaptureLoop(CancellationToken ct)
```

使用**自调度 async 循环**而非 `DispatcherTimer`，更健壮：

```csharp
while (!ct.IsCancellationRequested)
{
    await CaptureOneFrame(ct);
    int delay = Math.Max(50, 100 - processingMs);
    await Task.Delay(delay, ct);  // 动态调整帧率
}
```

每帧处理带 **1.5 秒超时保护**——防止 `CopyFromScreen` 在某些显卡配置下永久卡死：

```csharp
var captureTask = Task.Run(() => CaptureAndEnhance());
if (await Task.WhenAny(captureTask, Task.Delay(1500)) != captureTask)
{
    // 超时，跳过此帧
    return;
}
```

### 2.5 退出机制

通过 Win32 `RegisterHotKey` 注册全局 F11 热键，**无论焦点在哪个窗口**都能响应：

```csharp
RegisterHotKey(hwnd, HOTKEY_ID, 0, 0x7A); // VK_F11 = 0x7A
```

在 `WndProc` 中处理 `WM_HOTKEY` 消息。

## 三、录屏

### 3.1 帧捕获

复用 `DxgiScreenCapture` 类，与实时增强共享捕获逻辑。

### 3.2 帧缓存

```csharp
ScreenRecorder.cs
```

每帧捕获后立即保存为 **JPEG 文件**到临时目录：

```
/tmp/MediaEnhancerRec/{GUID}/f_000000.jpg
                                    f_000001.jpg
                                    ...
```

**为什么用 JPEG 而非内存中的 `List<byte[]>`？** 视频帧数据量大（1920×1080 ≈ 8MB/帧），60 秒 15FPS 视频约 900 帧 → 7.2GB 内存。JPEG 压缩后约 200KB/帧 → 约 180MB。

### 3.3 MP4 编码

#### 首选：WinRT MediaComposition

```csharp
Windows.Media.Editing.MediaComposition
```

将 JPEG 帧序列编码为 H.264 MP4：

```csharp
foreach (var fp in frameFiles)
{
    var file = await StorageFile.GetFileFromPathAsync(fp);
    composition.Clips.Add(await MediaClip.CreateFromImageFileAsync(file, duration));
}
await composition.RenderToFileAsync(outputFile);
```

**注意**：输出文件必须通过 `StorageFolder.CreateFileAsync` 创建，而非 `StorageFile.GetFileFromPathAsync`，否则桌面应用可能因权限不足失败。

#### 回退：帧序列目录

WinRT 编码可能因缺少包标识（Package Identity）失败。此时保存为 JPEG 帧序列文件夹，用户可自行用其他工具合成。

### 3.4 多线程架构

```
[UI 线程] 命令处理
    ↓ Start / Stop
[后台线程] RecordingLoop
    ├─ 帧捕获 (DXGI/GDI)
    ├─ 增强 (Enhance)
    ├─ 保存 JPEG
    └─ 进度通知
[UI 线程] UpdateRecordingDuration (timer)
    └─ 读取 DurationSeconds 更新界面
```

`RecordingLoop` 通过 `Task.Run` 在后台线程运行，不阻塞 UI。

## 五、关键经验

1. **全屏覆盖窗口** → 必须使用分层窗口（`AllowsTransparency=True`）+ `WS_EX_TRANSPARENT`，不能用 `IsHitTestVisible=False`
2. **递归消除** → `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)` 比"截屏后擦除自身区域"更干净
3. **`CopyFromScreen` 卡死** → 必须加超时保护（`Task.WhenAny` + `Task.Delay`）
4. **帧缓存** → 用 JPEG 而非内存，避免 OOM
5. **视频编码** → `MediaComposition` 需要 `StorageFolder.CreateFileAsync`；桌面应用可能需要回退方案
6. **MediaPlayer 渲染帧** → 必须调用 `Play()` 后帧才会实际渲染，仅设 `Position` 不行
