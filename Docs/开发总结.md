# 影音智增强系统 — 开发总结

## 项目概述

基于 .NET 10 WPF + MVVM 的 Windows 桌面影音管理应用，集成 AI 实时视觉增强、多模态大模型对话、AI 图像生成等功能。

| 项目 | 内容 |
|:---|:---|
| 框架 | .NET 10 WPF + CommunityToolkit.Mvvm |
| 数据库 | SQLite + EF Core，6 张表 |
| DI | Microsoft.Extensions.DependencyInjection |
| 代码规模 | ~10,000 行 C# + ~2,500 行 XAML |
| 目标用户 | 普通用户、安防人员、影音爱好者、视障群体 |

## 二、增强算法模块

### 接口架构

```
IEnhancementMethod                 ← 统一根接口
  ├── INativeEnhancement           ← 离线-传统算法（BitmapSource）
  │     └── LinearStretchMethod
  ├── IOnnxEnhancement             ← 离线-深度学习（预留）
  └── IRealTimeEnhancer            ← 实时逐帧（byte[]）
        └── LinearStretchMethod
```

### 接口重构

`IRealTimeEnhancer : IEnhancementMethod`（一行继承），将原本互不关联的三棵接口树合并为一。所有方法可通过 `EnhancementRegistry` 统一注册和切换。

### 线性拉伸算法

- C# 原生实现，纯 CPU，< 0.1ms/帧
- 参数：对比度强度（0.5~2.0）+ 亮度偏移（-50~50）
- 两遍扫描：直方图统计 → 线性映射到全动态范围
- 同时实现 `INativeEnhancement`（离线 `BitmapSource`）和 `IRealTimeEnhancer`（实时 `byte[]`）

## 三、实时全屏增强

### 技术栈

- **窗口**：分层窗口（`AllowsTransparency=True`）+ `WS_EX_TRANSPARENT` 鼠标穿透
- **捕获**：DXGI Desktop Duplication（GPU 零拷贝, 30-60 FPS, ~3% CPU）+ GDI 回退（5-10 FPS）
- **递归消除**：`SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)`
- **安全退出**：`RegisterHotKey` 注册 F11 全局热键
- **超时保护**：`Task.WhenAny` + `Task.Delay` 防止 `CopyFromScreen` 卡死

### DxgiScreenCapture 修复

- `CopyResource` 后添加 `ID3D11DeviceContext::Flush()`（vtable 54），消除 GPU 脏帧
- 区分 `DXGI_ERROR_WAIT_TIMEOUT`（0x887A0027）与真实错误，超时自动重试而非退回 GDI
- `CaptureFrame(maxWaitMs)` 支持自定义等待时长

## 四、屏幕录制模块

### 演进历程

| 迭代 | 方案 | 结果 |
|:---|:---|:---:|
| v1 | rawvideo stdin 管道 | ❌ ffmpeg 进程管理复杂 |
| v2 | concat 文件列表 | ❌ 路径转义 |
| v3 | image2pipe stdin | ❌ 与音频冲突 |
| v4 | `%08d` 图片序列 + 编码器逐级降级 | ✅ |
| v5 | 纯视频（去音频/麦克风）| ✅ 当前版本 |

### 当前架构

```
DXGI/GDI 捕获 → JPEG帧 → 输出目录
    ↓
ffmpeg -framerate {真实fps} -i %08d.jpg → MP4
编码器降级: h264_nvenc → h264_qsv → h264_amf → libx264 → mpeg4
成功 → 删帧目录；失败 → 保留帧目录
```

### 修复的 Bug

| 问题 | 修复 |
|:---|:---|
| D3D11 脏帧 | `CopyResource` 后加 `Flush` |
| 收藏勾选框 | `CellEditEnding` 事件 |
| `libx264` 缺失 | 编码器逐级降级，`mpeg4` 兜底 |
| `-shortest` 截断 0 帧 | 移除 `-shortest` |
| 音画不同步/时长不准 | 首尾帧时间戳反算真实帧率 |
| 录制字段硬编码 | `Duration`/`IsEnhanced` 写入真实值 |
| 标题不同步 | 绑定 `MediaFile.Title` + `.Include()` |

## 五、视频增强模块

### 流程

```
视频文件
  ├── ffmpeg 检测帧率
  ├── ffmpeg 解帧 → 临时目录
  ├── ffmpeg 提音轨 → audio.aac
  ├── Task.Run: 逐帧 IRealTimeEnhancer.Enhance(byte[])（后台线程）
  ├── ffmpeg 合帧 + 音轨 → MP4（编码器降级链）
  └── 自动入库
```

### 特性

- 帧率自适应：检测原视频 fps，合帧不减速
- 可取消：进度条旁「取消」按钮，CancellationToken 全链路
- 防并发：增强中禁止再次增强、实时增强、批量增强
- 后台跑：`Task.Run` + `byte[]` 路径，不卡 UI
- 进度提示：「正在增强视频... 45/300 帧」

## 六、AI 对话模块

### 模型

- Qwen3.7-Plus（阿里百炼），OpenAI 兼容 API
- 多模态：图片 base64 嵌入，视频 ffmpeg 抽关键帧

### 预设

| 预设 | 功能 | 降级行为 |
|:---|:---|:---|
| 📝 AI 简介 | 分析文件生成简介和标签 | 模板化分析文件名/时长/分辨率 |
| 📊 数据摘要 | 统计选中文件 | 本地计算（100% 可用） |
| 自由对话 | 多轮对话 + 追问 | 返回引导提示 |

### 降级链

```
API 未配置 → 模板化 AI → 标注「API 未配置」
API 调用失败 → 模板化 AI → 标注具体错误信息
API 正常 → 真实 LLM → 正常展示
```

### UI 特性

- 侧边栏文件勾选 + 预设按钮

- 输入框回车发送
- 等待时「🤔 正在思考」状态提示

## 七、AI 编辑模块

### API 格式支持

| 格式 | 行为 | 适用供应商 |
|:---|:---|:---|
| auto | URL 自动推断 | 默认 |

## 八、配置持久化

`appsettings.json` 统一管理所有配置：

```json
{
  "ChatEndpoint": "https://dashscope.aliyuncs.com/compatible-mode/v1",
  "ChatKey": "",
  "ChatModel": "qwen-plus",
  "EditEndpoint": "https://dashscope.aliyuncs.com/api/v1/services/aigc/image-generation/generation",
  "EditKey": "",
  "EditModel": "wanx2.0-t2i-turbo",
  "EditFormat": "auto",
  "RecordingPath": "D:\\...\\Recordings",
  "EnhancementPath": "D:\\...\\Enhancements",
  "ThumbnailPath": "D:\\...\\Thumbnails"
}
```

- AI 对话和 AI 编辑分开配置（Key + 端点 + 模型）
- 三个保存路径修改即时持久化
- 启动时自动加载
