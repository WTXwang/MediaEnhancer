## 一、项目根目录

### App.xaml / App.xaml.cs — 应用程序入口

**文件路径**：`App.xaml`、`App.xaml.cs`

**包含的类**：`App`（继承 `Application`）

**主要作用**：WPF 应用程序的启动入口，负责依赖注入容器的构建、服务的注册、增强算法的注册、数据库迁移的触发以及登录流程的控制。

**代码解释**：

1. **`OnStartup` 方法**（第19-78行）：应用启动的核心流程
   - 设置 `ShutdownMode = OnExplicitShutdown`，防止登录窗口关闭后自动退出整个应用
   - 调用 `ConfigureServices` 构建 DI 容器
   - 向 `EnhancementRegistry` 注册增强方法：`LinearStretchMethod`（纯 C#，始终可用）和 `MultinexNanoMethod`（ONNX，加载失败静默跳过）
   - 实例化 `AppDbContext` 触发 `Database.Migrate()` 自动建表/迁移
   - 显示登录窗口并阻塞等待：`loginWindow.ShowDialog()`
   - 登录成功后显示主窗口，启动后台缩略图缓存清理任务

2. **`ConfigureServices` 方法**（第81-110行）：依赖注入注册
   - 数据库：`AddDbContextFactory<AppDbContext>` 配合 SQLite，支持 `AuthService` 等独立创建 context 的场景
   - 单例：`EnhancementRegistry`、`AiService`、`IAuthService`
   - Scoped：`IDataService`、`IFileScanService`、`IPlaybackService`、`IThumbnailService`
   - Transient：`MainViewModel`、`LoginViewModel`、`MainWindow`、`LoginWindow`

## 二、Core 核心层

### Core/IEnhancementMethod.cs — 增强方法根接口

**包含的类/接口**：`IEnhancementMethod`（接口）

**主要作用**：定义所有增强方法的共同契约——三个只读属性：
- `Name`：方法名称
- `Description`：方法描述
- `Parameters`：可调参数列表（`List<EnhancementParameter>`）

**代码解释**：这是整个增强架构的基石。所有增强方法（无论是实时还是离线、原生还是 ONNX）都必须间接实现此接口。`EnhancementRegistry.GetMethod()` 的返回类型即为此接口，使得 UI 层可以统一获取任何增强方法的基本元数据。

---

### Core/IRealTimeEnhancer.cs — 实时增强接口

**包含的类/接口**：`IRealTimeEnhancer`（接口，继承 `IEnhancementMethod`）

**主要作用**：定义 BGRA32 原始字节数组级别的逐帧增强契约。消费者（如 `FullscreenEnhanceWindow`）只依赖此接口，不依赖具体算法。

**代码解释**：
- `SupportsRealTime` 属性：标识方法是否能跟上帧率（<33ms/帧可达30FPS），为 false 的方法不会出现在实时增强下拉列表中
- `Enhance(byte[] pixels, int width, int height, int stride, IReadOnlyDictionary<string, double>? parameters)` 方法：接收原始 BGRA32 像素数据（每像素4字节，B→G→R→A 顺序），返回同格式同尺寸的增强结果。`stride` 参数支持行对齐（通常为 width×4，但可能有更大的对齐值）

---

### Core/INativeEnhancement.cs — 原生增强接口

**包含的类/接口**：`INativeEnhancement`（接口，继承 `IEnhancementMethod`）

**主要作用**：定义基于 WPF `BitmapSource` 的离线同步增强契约，适用于线性拉伸等纯 C# 算法。

**代码解释**：`Enhance(BitmapSource input) → BitmapSource` 方法——接收 WPF 位图对象，返回增强后的位图对象。这个路径用于离线文件增强场景，因为 WPF `BitmapSource` 适合单张图片的完整处理流程。

---

### Core/IOnnxEnhancement.cs — ONNX 增强接口

**包含的类/接口**：`IOnnxEnhancement`（接口，继承 `IEnhancementMethod`）

**主要作用**：定义 ONNX 深度学习模型增强的异步契约，支持模型文件管理和 CPU/GPU 推理。

**代码解释**：
- `ModelFileName`：模型文件名（不含路径），文件位于运行目录的 `OnnxModels/` 子目录下
- `MaxInferenceSize`：推理尺寸上限（长边像素数），超此值的输入先缩小再放大。实时场景可设为480以速度换质量，离线场景设为0获得最佳画质
- `EnhanceAsync(BitmapSource input)`：使用内部缓存的模型进行异步推理
- `EnhanceAsync(BitmapSource input, string modelPath)`：使用指定路径的外部模型文件（每次新建 Session，用后即弃）

---

### Core/EnhancementParameter.cs — 增强参数模型

**包含的类**：`EnhancementParameter`（普通类）

**主要作用**：定义增强算法的单个可调参数，含名称、键名、当前值、范围（最小值/最大值）和步进。UI 层读取 `List<EnhancementParameter>` 自动生成滑块控件。

**代码解释**：五个属性全部为 `{ get; set; }` 自动属性：
- `Name`：参数的 UI 显示名称（如"对比度强度"）
- `Key`：参数的内部键名（如"contrast"），用于传递到算法的参数字典中
- `Value`：当前值
- `Min`、`Max`：取值范围
- `Step`：滑块的步进精度

---

### Core/EnhancementRegistry.cs — 增强方法注册中心

**包含的类**：`EnhancementRegistry`（普通类，单例注入）

**主要作用**：管理所有 `IRealTimeEnhancer` 和 `IOnnxEnhancement` 的注册、切换与枚举。消费者（全屏窗口、录屏器、ViewModel）通过此注册中心获取当前算法，不直接依赖具体实现。

**代码解释**：

1. **内部存储**：两个 `Dictionary<string, T>` 分别存储实时方法和离线方法
   - `_methods: Dictionary<string, IRealTimeEnhancer>` —— 实时方法
   - `_offlineMethods: Dictionary<string, IOnnxEnhancement>` —— 离线 ONNX 方法
   - `_current: IRealTimeEnhancer?` —— 当前选中的实时方法

2. **`Register(IRealTimeEnhancer method)`**：将方法加入实时字典，第一个注册的方法自动成为 `_current`

3. **`RegisterOffline(IOnnxEnhancement method)`**：将方法加入离线字典

4. **`SetCurrent(string methodName)`**：切换到指定名称的实时方法，返回是否成功

5. **`GetMethod(string methodName)`**：按名称查找（先实时后离线），返回 `IEnhancementMethod?`，供 UI 获取方法元数据

6. **属性暴露**：`RealTimeMethodNames`、`OfflineMethodNames`、`MethodNames`（合并）、`CurrentMethodName`——供 ViewModel 的下拉列表绑定

### Core/LinearStretchMethod.cs — 线性拉伸增强算法

**包含的类**：`LinearStretchMethod`（实现 `INativeEnhancement` 和 `IRealTimeEnhancer`）

**主要作用**：纯 C# 实现的直方图拉伸增强算法，零外部依赖，同时支持实时逐帧增强（`byte[]` 路径）和离线图片增强（`BitmapSource` 路径）。

**代码解释**：

**参数定义**（第53-73行）：
- `contrast`：对比度强度（0.5~2.0，默认 1.0）
- `brightness`：亮度偏移（-50~+50，默认 0）

**实时路径 `Enhance(byte[], ...)`**（第102-185行）——两遍扫描算法：
1. **第一遍——统计亮度直方图**（第120-133行）：遍历所有像素，使用 ITU-R BT.601 标准 luma 权重（R×0.299 + G×0.587 + B×0.114）计算亮度值，建立 256 级直方图
2. **确定有效亮度范围**（第138-158行）：分别从暗端和亮端向中间扫描，裁剪首尾各 2% 的极端像素（去除噪声和过曝），找到 [minVal, maxVal]
3. **均匀图像保护**（第161行）：如果 maxVal ≤ minVal（全图同色），重置为 [0, 255] 避免除以零
4. **第二遍——线性映射**（第163-184行）：使用 `Buffer.BlockCopy` 复制输入数组（避免修改原数据），对 B/G/R 三个通道应用公式 `new = (old - minVal) × (255/range) × contrast + brightness`，然后钳位到 [0,255]，Alpha 通道保持不变

**离线路径 `Enhance(BitmapSource)`**（第192-262行）：
- 转换为 `WriteableBitmap` 以使用 unsafe 指针直接操作像素内存
- 算法逻辑同上，但操作的是 `byte*` 指针而非托管数组，性能更高
- 处理完毕后调用 `AddDirtyRect` 标记变更区域，`Unlock` 提交修改

---

### Core/MultinexNanoMethod.cs — ONNX 深度学习增强

**包含的类**：`MultinexNanoMethod`（实现 `IRealTimeEnhancer` 和 `IOnnxEnhancement`，同时实现 `IDisposable`）

**主要作用**：基于 ONNX Runtime 的超轻量 Retinex 低光照增强模型（15K 参数），提供实时和离线两条增强路径。

**代码解释**：

1. **延迟加载 + 双重检查锁定**（`EnsureSession`，第115-124行）：
   - 首次调用 `Enhance` 时才从 `OnnxModels/` 目录加载模型文件
   - 双重检查锁定确保多线程安全：快速路径（已加载直接返回）→ lock → 再次检查 → 创建 `InferenceSession`
   - 后续调用复用缓存的 Session，避免重复加载

2. **实时路径 `Enhance(byte[], ...)`**（第45-56行）：
   - `EnsureSession()` 确保模型已加载
   - `OnnxModelHelper.Preprocess`：BGRA32 bytes → NCHW float32（maxSize=0 全分辨率）
   - `OnnxModelHelper.RunInference`：送入 ONNX 模型推理
   - `OnnxModelHelper.Postprocess`：NCHW float32 → BGRA32 bytes

3. **离线路径 `EnhanceAsync(BitmapSource)`**（第71-86行）：
   - 在 UI 线程读取 `BitmapSource` 像素数据
   - 推理计算在 `Task.Run` 中异步执行，避免阻塞 UI

4. **外部模型路径 `EnhanceAsync(BitmapSource, string modelPath)`**（第92-104行）：
   - 每次创建独立的 `InferenceSession`，用完即释放，适合测试不同权重文件

---

### Core/OnnxModelHelper.cs — ONNX 预处理/后处理工具

**包含的类**：`OnnxModelHelper`（静态类）

**主要作用**：在 WPF 图像格式（BGRA32 `byte[]` 或 `BitmapSource`）和 ONNX 模型格式（NCHW float32 [0,1]）之间做双向转换，并提供 Bilinear 缩放和推理封装。

**代码解释**：

1. **降采样尺寸计算 `GetInferenceSize`**（第33-39行）：
   - maxSize ≤ 0 或输入已在限制内 → 返回原始尺寸
   - 否则按长边等比缩放，保持宽高比

2. **双线性插值缩放 `ResizeBilinear`**（第47-89行）：
   - 对每个目标像素，找到源图中的 4 个相邻像素（左上、右上、左下、右下）
   - 先按水平权重 fx 混合左右邻居，再按垂直权重 fy 混合上下结果
   - 对 B/G/R/A 四个通道分别插值

3. **`Preprocess(byte[], ...)`**（第109-141行）——BGRA32 → NCHW float32：
   - 如果设置了降采样，先通过 `ResizeBilinear` 缩小
   - 内存布局变换：`BGRA[srcIdx+2]=R → NCHW[0][y][x]`、`BGRA[srcIdx+1]=G → NCHW[1][y][x]`、`BGRA[srcIdx+0]=B → NCHW[2][y][x]`
   - 像素值除以 255 归一化到 [0, 1]

4. **`Postprocess` + `PostprocessDirect`**（第147-180行）——NCHW float32 → BGRA32：
   - 逆映射：`NCHW[0]=R → BGRA[+2]`、`NCHW[1]=G → BGRA[+1]`、`NCHW[2]=B → BGRA[+0]`
   - Alpha 通道固定为 255（不透明白色）
   - 输出值通过 `ClampToByte` 钳位到 [0, 255]

5. **`RunInference`**（第253-268行）：
   - 创建 4D `DenseTensor<float>`（shape [1, 3, H, W]）
   - 输入名为固定 "input"
   - 取 `session.Run()` 的第一个输出 tensor，转为数组返回

6. **`Preprocess(BitmapSource)` 和 `Postprocess(float[], int, int)`**（第189-244行）：
   - BitmapSource 路径的对应版本，内部逻辑相同，但输入输出类型不同
   - Postprocess 生成的 `BitmapSource` 会调用 `Freeze()` 使其跨线程安全

**主要作用**：将 AI 对话返回的 Markdown 文本转换为 WPF `FlowDocumentScrollViewer`，支持文本选择/复制。

## 三、Models 数据模型层

### Models/User.cs — 用户实体

**包含的类**：`User`

**主要作用**：对应 SQLite 中的 `Users` 表，存储用户认证信息。密码以 SHA256(salt + password) 哈希形式存储。

| 属性 | 类型 | 说明 |
|------|------|------|
| `Id` | `int` | 主键，自增 |
| `Username` | `string` | 登录用户名（唯一索引） |
| `PasswordHash` | `string` | SHA256 密码哈希 |
| `Salt` | `string` | 随机盐值（Base64，32字节） |
| `DisplayName` | `string` | 界面显示名称 |
| `CreatedAt` | `DateTime` | 账号创建时间 |
| `LastLoginAt` | `DateTime?` | 末次登录时间 |

### Models/MediaFile.cs — 媒体文件实体

**包含的类**：`MediaFile`

**主要作用**：对应 `MediaFiles` 表，是系统的核心实体，存储导入的图片/视频/音频文件的所有元数据。

| 属性 | 类型 | 说明 |
|------|------|------|
| `Id` | `int` | 主键，自增 |

### Services/DxgiScreenCapture.cs — DXGI 屏幕捕获

**包含的类**：`DxgiScreenCapture`（实现 `IDisposable`）

**主要作用**：基于 DXGI Desktop Duplication API 的纯 P/Invoke 屏幕捕获（零第三方依赖），支持 Windows 8.1+，比 GDI `CopyFromScreen` 更高效。

**代码解释**：

1. **`Initialize()` 初始化流程**（第201-273行）：共 8 步——
   - 创建 DXGI Factory
   - 枚举第一个显卡适配器

### Services/ScreenRecorder.cs — 屏幕录制器

**包含的类**：`ScreenRecorder`（实现 `IDisposable`）

**主要作用**：完整的屏幕录制流水线——逐帧 JPEG 保存 → 后处理增强（可选）→ FFmpeg 编码 MP4。关键设计：录制时不增强（保证帧率），停止后才逐帧增强。

**代码解释**：

1. **`Start()`**（第69-83行）：创建帧缓存目录 `rec_{timestamp}_frames`，启动后台 `RecordingLoop` Task

2. **`RecordingLoop` 录制循环**（第218-286行）：
   - 优先 DXGI（`DxgiScreenCapture`），失败回退 GDI（`Graphics.CopyFromScreen`）
   - 每帧存为 JPEG（`SaveAsJpeg` 使用 GDI `Bitmap.LockBits` + `Marshal.Copy`）
   - 帧率控制：计算剩余时间，>2ms 才 `Task.Delay`（避免微秒级 sleep 的调度开销）

3. **`StopAsync` 停止流程**（第85-137行）：
   - 取消录制循环
   - 有增强器 → `EnhanceFrames` 后台逐帧增强（优先 ONNX、回退实时方法）
   - `TryFfmpegAsync` 编码：五级降级链 `h264_nvenc → h264_qsv → h264_amf → libx264 → mpeg4`
   - 成功 → 删帧目录，返回 MP4 路径；失败 → 保留帧目录

4. **`EnhanceFrames` 后处理增强**（第144-212行）：
   - 遍历所有 `*.jpg` 帧文件
   - 优先 `_offlineEnhancer.EnhanceAsync`（全分辨率 ONNX），回退 `_enhancer.Enhance`（线性拉伸等）
   - 使用 WPF `BitmapImage` + `JpegBitmapEncoder` 避免 GDI+ ExternalException

**主要作用**：四阶段视频增强管线——FFmpeg 解帧 → 提取音轨 → 逐帧增强 → 合帧+音轨。

2. **页面导航**：`SelectedPageIndex` (0-7) 控制右侧面板切换，`ChangePageCommand` 由侧边栏按钮触发，`OnSelectedPageIndexChanged` 钩子在进入离线增强/实时增强页面时自动刷新对应记录

3. **文件管理**：
   - 三维过滤：`SearchText`（关键词）+ `FilterType`（类型）+ `FilterFavoritesOnly`（收藏）
   - `LoadWithCancel()` 取消上一次加载请求，防止快速输入时旧结果覆盖新结果
   - `Refresh` 命令：遍历所有文件，自动修复缺失信息（删除失效记录、补充分辨率、生成缩略图）
   - `ImportFilePathsAsync`：统一导入逻辑，被 ImportFiles、EnhanceFile、ExportEnhanced、录屏完成等多个流程复用
   - `DeleteFile`：弹出 `DeleteConfirmDialog` 三种选择（取消/仅删记录/删源文件）
   - `RenameFile`：`InputDialog` 输入新名称 → `File.Move` 文件系统重命名 → 更新数据库
   - `EnsureFileExistsAsync`：源文件缺失时的三方对话框（删除记录/手动定位/取消）

4. **批量操作**：`BatchToggleFavorite`（多数决定）、`BatchDelete`、`BatchGenerateThumbnails`

### Views/FullscreenEnhanceWindow.xaml.cs — 全屏增强窗口

**包含的类**：`FullscreenEnhanceWindow`（继承 `Window`）

**主要作用**：全屏透明覆盖窗口，捕获屏幕内容 → 实时增强 → 透明叠加显示。使用 DXGI Desktop Duplication + GDI 自动回退、鼠标穿透、F11 热键退出。

**代码解释**：

1. **窗口初始化**（第73-103行）：
   - 设置窗口尺寸为 `SystemParameters.PrimaryScreenWidth/Height`（WPF 逻辑像素）
   - 注册 `SourceInitialized` 事件（窗口句柄可用后执行 P/Invoke 设置）

2. **`OnSourceInitialized`**（第105-123行）：
   - 获取 `HwndSource` 句柄
   - `SetWindowDisplayAffinity(WDA_EXCLUDEFROMCAPTURE)` 消除画中画递归
   - 注册 F11 全局热键

3. **`Start(IRealTimeEnhancer, parameters)`**（第138-189行）：
   - 获取 DPI 缩放因子，计算物理像素分辨率
   - 初始化 DXGI 捕获（失败自动回退 GDI）
   - 设置 `WS_EX_TRANSPARENT` 鼠标穿透
   - 启动 `CaptureLoop`

4. **`CaptureOneFrame`**（第217-292行）——核心帧循环：
   - DXGI 优先捕获一帧，失败回退 GDI
   - 调用 `_method.Enhance(byte[])` 逐帧增强
   - 创建 `BitmapSource` 时使用物理 DPI（`96 × scale`），实现 1:1 像素映射
   - `Dispatcher.Invoke` 回 UI 线程更新显示
   - 2 秒超时保护防止 `CopyFromScreen` 卡死
